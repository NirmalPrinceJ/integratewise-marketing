# IntegrateWise Documentation System — Master Index

## Purpose

This document defines the full documentation system for IntegrateWise as a category-defining product and company. It organizes all required documentation into six master packs, clarifies priorities, and gives a practical creation sequence.

IntegrateWise is not only a software product. It is a Knowledge Workspace empowered by AI and the Spine, with governance, approvals, connectors, context intelligence, and enterprise operating-model implications. Because of that, its documentation system must cover company, market category, product, architecture, trust, and go-to-market.

## Final Principle

IntegrateWise should not treat documentation as isolated files. The documentation system itself should reflect the product strategy:

* truth should be canonical
* language should be consistent
* architecture should map to narrative
* governance should be explicit
* GTM materials should inherit from the same source foundations

That way, every deck, page, brochure, architecture paper, and product document tells the same story from a different altitude.
